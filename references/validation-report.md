## Independent static smoke simulation

An independent read-only reviewer simulated ten routes against the authored instructions. Initial review found ambiguous planning-only/hotfix/release-evidence/operational-documentation routes. Those findings were fixed and the reviewer reran the current files.

Final result: 17/17 defined scenarios pass. The original ten lifecycle routes still pass, and seven curated-redesign routes were added.

1. Clarify and plan a small bug: compact planning-only route, bug evidence/test/rollback, no edits.
2. Design and implement a medium feature: full gated lifecycle and actual-evidence requirements.
3. Break down a large initiative: milestones, critical path, RAID, multi-PR/release slices, no edits under planning authority.
4. Review a PR for release readiness: exact scope, tests/CI, findings, security/operations, rollout/rollback.
5. Prepare a completed change for release: missing-evidence repair loop and return to release gate.
6. Document an operational change: canonical docs plus security/operations/release readiness.
7. Design-only: decision-ready design/ADR, explicit stop.
8. Review-only: evidence-backed findings, explicit no-edit boundary.
9. Documentation-only: canonical update and appropriate docs validation.
10. Incident/hotfix: bounded root-cause fix, regression evidence, authorized rollout/rollback, verification, retrospective.
11. Meeting notes to work: source freshness, confirmed decisions/action provenance, preview-before-write, exact targets, receipts or drafts.
12. Cross-system reconciliation: canonical source map, native/normalized states, contradictions, freshness, and reconciliation.
13. Security candidate closure: pinned scope/coverage, proof/counterevidence, stable findings, suppressed/deferred states.
14. CI/provider fallback: first causal failure, native/external distinction, same-revision evidence, narrow retry or explicit gap.
15. Flagged failed deployment: release unit/environment, flag lifecycle, telemetry baseline, state/rollback, incident timeline.
16. Preview plus migration: isolation/parity/TTL/teardown, compatibility, restore evidence, and artifact identity.
17. Platform evidence: focused scenario, exact runtime/build/revision, fresh comparable artifacts, and distribution evidence.

Cross-cutting checks passed: scale-down/scale-up behavior, no silent gate deletion, epistemic labels, evidence truthfulness, optional connectors, no legacy runtime dependency, external write/readback boundaries, provider-neutral CI/release evidence, and authority boundaries for editing/merging/publishing/deploying/destructive actions.

## Residual limitations and release status

This evidence supports a validated local release candidate, not a claim that every downstream repository or production system is proven. Static simulation does not execute a real target repository’s tests, CI, PR checks, migrations, deployments, or production verification. Before uninstalling superseded plugins, start a new Codex task, run live smoke prompts on representative repositories, and complete the disable-one-at-a-time canary gates in `migration-and-decommission.md`.
